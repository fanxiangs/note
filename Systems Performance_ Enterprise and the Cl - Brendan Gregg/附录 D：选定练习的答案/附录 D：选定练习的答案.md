## 第7章——记忆

问：使用 Linux 术语，分页和交换之间有什么区别？

A. 分页是内存页的移动；交换是页面往返于交换设备/文件的移动。

问：描述内存利用率和饱和度。

答：对于内存容量，利用率是指根据可用内存总量衡量的正在使用且不可用的数量。这可以用百分比表示，类似于文件系统容量。饱和度是对超出内存大小的可用内存需求的度量，通常会调用内核例程来释放内存以满足此需求。

## 第 8 章—文件系统

问：逻辑 I/O 和物理 I/O 有什么区别？

A.逻辑I/O是到文件系统接口；物理 I/O 是针对存储设备（磁盘）的。

问：解释文件系统写时复制如何提高性能。

答：由于随机写入可以写入新位置，因此可以将它们分组（通过增加 I/O 大小）并按顺序写出。这两个因素通常都会提高性能，具体取决于存储设备类型。

## 第 9 章—磁盘

问：请描述当磁盘工作超载时会发生什么情况，包括对应用程序性能的影响。

A. 磁盘持续高利用率（高达100%）运行，且饱和度高（排队）。由于排队的可能性（可以建模），它们的 I/O 延迟会增加。如果应用程序正在执行文件系统或磁盘 I/O，则增加的延迟可能会损害应用程序性能，前提是它是同步 I/O 类型：读取或同步写入。它还必须发生在关键应用程序代码路径期间，例如在处理请求时，而不是异步后台任务（这可能只会间接导致应用程序性能不佳）。通常，由于 I/O 延迟增加而产生的背压会控制 I/O 请求的速率，并且不会导致延迟无限增加。

## 第 11 章—云计算

问：描述来自操作系统虚拟化客户的物理系统的可观察性。

答：根据主机内核实现，来宾可以查看所有物理资源（包括 CPU 和磁盘）的高级指标，并注意到其他租户何时使用这些资源。泄漏用户数据的指标应该被内核阻止。例如，CPU 的利用率可能是可观察的（例如 50%），但无法观察到导致该利用率的其他租户的进程 ID 和进程名称。
