#### GPU

图形处理单元 (GPU) 旨在支持图形显示，现在可用于其他工作负载，包括人工智能、机器学习、分析、图像处理和加密货币挖掘。对于服务器和云实例，GPU 是一种类似处理器的资源，可以执行一部分工作负载（称为计算内核），适合高度并行的数据处理，例如矩阵转换。Nvidia 使用统一计算设备架构 (CUDA) 的通用 GPU 已得到广泛采用。CUDA 提供用于使用 Nvidia GPU 的 API 和软件库。

虽然处理器 (CPU) 可能包含十几个核心，但 GPU 可能包含数百或数千个称为流处理器 (SP) 的较小核心，[^5] 每个核心都可以执行一个线程。由于 GPU 工作负载是高度并行的，因此可以并行执行的线程被分组为线程块，它们可以在线程块中相互协作。这些线程块可以由称为流式多处理器 (SM) 的 SP 组执行，这些处理器还提供包括内存高速缓存在内的其他资源。表 6.6 进一步比较了处理器 (CPU) 和 GPU \[Ather 19\]。

[^5]: Nvidia 也将这些 CUDA 核心称为 \[Verma 20\]。

### 表 6.6 CPU 与 GPU 的比较

| 属性       | 中央处理器 (CPU)                                | 图形处理器 (GPU)                                      |
|------------|------------------------------------------------|------------------------------------------------------|
| **包裹**   | 处理器包插入系统板上的插槽，直接连接到系统总线或 CPU 互连。 | GPU 通常作为扩展卡提供并通过扩展总线（例如 PCIe）连接。它们还可以嵌入系统板或处理器封装（片上）中。 |
| **封装可扩展性** | 多插槽配置，通过 CPU 互连（例如 Intel UPI）连接。      | 多 GPU 配置是可能的，通过 GPU 到 GPU 互连（例如 NVIDIA 的 NVLink）进行连接。 |
| **核心**   | 当今的典型处理器包含 2 至 64 个内核。            | GPU 可能具有类似数量的流式多处理器 (SM)。               |
| **线程数** | 典型的核心可以执行两个硬件线程（或更多，取决于处理器）。   | SM 可能包含数十或数百个流处理器 (SP)。每个 SP 只能执行一个线程。 |
| **缓存**   | 每个核心都有 L2 和 L2 缓存，并且可以共享 L3 缓存。   | 每个 SM 都有一个缓存，并且可以在它们之间共享 L2 缓存。    |
| **时钟**   | 高（例如 3.4 GHz）。                            | 相对较低（例如 1.0 GHz）。                              |

必须使用自定义工具来实现 GPU 可观察性。可能的 GPU 性能指标包括每个周期的指令数、缓存命中率和内存总线利用率。
